# AI Content Detector 🔍

纯统计分析版 AI 写作检测器，无需 API 调用，零成本运行。

## 快速开始

```bash
# 直接运行
node index.js "你的文本内容"

# 从文件读取
node index.js /path/to/file.txt

# 测试模式（内置样例）
node index.js --test
```

## 核心指标

| 指标 | 说明 | AI 特征 |
|------|------|---------|
| **Burstiness** | 句子长度变化性 | 低（句子均匀） |
| **TTR** | 词汇丰富度 | 低（词汇重复） |
| **标点模式** | 标点密度和多样性 | 规范、稳定 |
| **段落结构** | 段落长度变化 | 均匀 |
| **AI 句式模板** | 典型 AI 句式关键词 | 高密度 |

## 技术原理

### 1. Burstiness（突发性）

- **计算公式**: `burstiness = sqrt(variance) / mean`
- **原理**: AI 文本句子长度方差小，变化平稳；人类文本长短交替，波动大
- **阈值**: burstiness < 0.3 → AI；burstiness > 0.5 → 人类

### 2. TTR（词汇丰富度）

- **计算公式**: `TTR = unique_words / total_words`
- **原理**: AI 词汇重复率高，TTR 较低；人类词汇多样，TTR 较高
- **阈值**: TTR < 0.4 → AI；TTR > 0.5 → 人类

### 3. AI 句式模板检测

检测常见 AI 写作模板关键词：
- 首先、其次、然后、最后
- 总之、综上所述
- 值得注意的是
- 一方面、另一方面
- 具体来说、换句话说

**权重**: 检测到 4+ 个模板关键词 → 98% AI 可能性

## 准确度

基于测试样例：
- **AI 文本检测**: ~76% 准确度
- **人类文本检测**: ~38% AI 占比（正确判断）

相比 GPTZero（85-90%），本工具为纯统计分析版本，适合快速筛查。建议结合人工判断。

## 局限性

1. **短文本不可靠**: <50 字检测结果不稳定
2. **混合内容困难**: 人类改写 AI 内容难以判断
3. **中文优化**: 当前针对中文文本优化，英文需调整参数
