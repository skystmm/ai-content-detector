# AI Content Detector Skill (v2)

> 整合 Humanizer-zh 规则，基于维基百科"AI写作特征"指南

## 功能定位

根据 Harrison Chase 持续学习框架，第一步是**先把 traces 做对**。

本 Skill 实现：
1. **AI率检测**：24种AI写作模式识别 + 朱雀官网验证（双保险）
2. **Traces记录**：记录检测历史，便于追溯和分析
3. **改写建议**：基于 Humanizer-zh 的具体改写前后示例
4. **质量评分**：50分制量化评估

**更新日志**：
- 2026-05-04：整合 Humanizer-zh 24种AI写作模式，添加质量评分体系
- 2026-04-13：新增模型辅助预判，减少手动检测次数

---

## 核心工作流

### Step 1：接收输入

支持两种输入方式：
- **文件路径**：读取指定文件，提取正文（不含 YAML 元数据）
- **直接内容**：用户粘贴的文章内容

**优先级**：如果检测到文件路径，优先读取文件；否则使用粘贴内容。

---

### Step 2：24种AI写作模式识别（v2）

使用 LLM 进行深度分析，基于维基百科"AI写作特征"指南识别 24 种模式。

**检测 prompt**（见下方 scripts/detect-prompt.md）

**输出格式（v2）**：

```markdown
## 🤖 AI内容检测报告 (v2)

### 整体评估
- **预估AI率**：XX%
- **风险等级**：🟢 低 / 🟡 中 / 🔴 高
- **质量评分**：XX/50
- **核心问题**：[一句话概括]

### 📌 检测到的AI模式

| 模式编号 | 模式名称 | 严重程度 | 示例词汇 |
|---------|---------|---------|---------|
| P7 | AI词汇过度使用 | 高 | 此外、至关重要、强调 |
| P1 | 夸大象征意义 | 中 | 标志着、见证了 |
| P10 | 三段式法则 | 低 | 强行分成三组 |

### 高风险段落（含改写前后示例）

#### 第3段
> 原文片段："新的软件更新作为公司致力于创新的证明..."

**匹配模式**：P1（夸大象征意义）、P7（AI词汇）

**改写前**：
> 新的软件更新作为公司致力于创新的证明。此外，它提供了无缝、直观和强大的用户体验。

**改写后**：
> 软件更新添加了批处理、键盘快捷键和离线模式。

**改写理由**：
- 删除"作为……的证明"（夸大象征）
- 删除"此外"（AI词汇）
- 删除"无缝、直观和强大"（三段式宣传）

### 低风险段落
- 第1段、第5段（保留技术术语原文，节奏自然）

### 改写优先级
1. **第3段** - P1+P7，AI率最高
2. **第7段** - P9（否定式排比）

### 💡 Humanizer 核心规则建议
- **删除填充短语**："值得注意的是数据显示" → "数据显示"
- **打破公式结构**：删除二元对比、三段式
- **变化节奏**：长短句交错，避免结尾雷同
- **信任读者**：直接陈述，跳过手把手引导
```

---

### Step 3：人工复核（可选）

根据模型预判结果，选择：
- **AI率 < 30%**：直接通过，记录 trace
- **AI率 30-50%**：建议人工改写后重新检测
- **AI率 ≥ 50%**：
  - 可选用朱雀官网二次验证
  - 或直接根据模型建议改写

朱雀检测指引（仅在需要时使用）：

```
检测工具：朱雀 AI 检测
官网地址：https://matrix.tencent.com/ai-detect/
```

---

### Step 4：阈值判断与建议

| AI率 | 状态 | 建议 |
|------|------|------|
| < 30% | ✅ 通过 | 记录 trace，继续流程 |
| 30-50% | ⚠️ 警告 | 根据模型建议改写，重新检测 |
| ≥ 50% | ❌ 不通过 | 必须大改，建议朱雀二次验证 |

---

### Step 5：记录 Traces

无论通过还是不通过，都记录检测历史，便于持续学习。

**traces 格式**：

```markdown
---
date: 2026-04-12
article: 文章标题（或内容前50字）
source: 文件路径 或 直接粘贴
tool: 模型辅助检测 + 朱雀验证
ai_rate: 45%
threshold: 30%
status: ⚠️ 警告
---

## 检测内容摘要
[文章核心内容摘要]

## 检测结果
- AI率：45%
- 判断：⚠️ 警告（需改写）

## 高风险段落
- 第3段："总而言之..."
- 第7段："值得注意的是..."

## 改写建议
- 第3段 → 替换为个人观点表达
- 第7段 → 删除过渡词

## 下一步行动
- 人工改写高 AI率段落
- 重新检测
```

**存储位置**：`~/.openclaw/workspace/ai-detection-traces/`
**文件命名**：`trace-{标题/日期}-{时间}.md`

## 朱雀检测实战经验（2026-04-19 新增）

### 核心发现：朱雀判定逻辑与直觉相反

**实测案例**："程序员学习新范式"文章改写历程

| 改写次数 | AI特征 | 改写策略 | 效果 |
|---------|--------|---------|------|
| 第一次 | 48.01% | 原文 | 基线 |
| 第二次 | 76.54% | 过度简化 | ❌ 变差 |
| 第三次 | 89.57% | 刻意口语化 | ❌❌ 更差 |
| 第四次 | 55.47% | 策略反转 | ✅ 有效 |

---

### ❌ 错误策略（会导致AI特征上升）

**错误一：过度简化**
- 删除内容、简化表达 → AI特征上升
- 朱雀判定：过于完美的简洁 = AI特征

**错误二：刻意口语化**
以下表达是 **AI模仿人类的典型特征**：
- "说实话..."
- "其实..."
- "读到这的时候..."
- "让我松了一口气..."
- "我挺期待..."
- 完美的口语化过渡词
- 刻意的情感表达

**朱雀判定逻辑**：
- 完美流畅的口语化 = AI生成
- 刻意加入的"人类感" = AI模仿人类

---

### ✅ 正确策略（会降低AI特征）

**正确一：保留技术术语**
- 直接使用英文原文术语：Loop until done、Git is memory
- 不翻译、不解释、不过渡

**正确二：保持不完美表达**
- 跳跃、简短、不连贯的表达
- 不要追求"流畅"
- 不要追求"口语化过渡"

**正确三：具体细节**
- 具体文件名、具体行数、具体错误
- 但不要具体到编造细节（保持真实性）

**正确四：删除整齐结构**
- 删除并列句式
- 删除"传统方式 vs 新方式"对比框
- 删除加粗的并列列表

---

### 改写流程优化

**Step 1：朱雀检测**
- 使用朱雀官网检测原文
- 记录：人工特征、疑似AI、AI特征

**Step 2：判断改写方向**
| AI特征来源 | 改写方向 |
|-----------|---------|
| 过度简化（人工特征低） | 增加具体细节、保留技术术语 |
| 刻意口语化（疑似AI高） | 删除口语化过渡词、保持简洁 |
| 整齐结构（AI特征高） | 打破对称、删除并列 |

**Step 3：避免常见错误**
- ❌ 不要加"说实话"、"读到这的时候"
- ❌ 不要追求流畅口语化
- ❌ 不要过度删减内容
- ✅ 保留技术术语原文
- ✅ 保持不完美表达
- ✅ 删除整齐并列结构

**Step 4：验证改写效果**
- 每次改写后用朱雀验证
- 观察"人工特征"是否上升
- 目标：AI特征 < 30%，人工特征 > 20%

---

### 人工特征 vs AI特征判断逻辑

**朱雀判定"人工特征"的标准**：
- 具体细节（真实的项目名、文件名）
- 不完美表达（跳跃、简短）
- 技术术语的直接使用
- 没有刻意过渡词

**朱雀判定"AI特征"的标准**：
- 完美的流畅度
- 刻意的口语化过渡
- 整齐的并列结构
- 抽象的感受表达

---

### 实战改写示例

**错误改写**（AI特征上升）：
```
原文：学习没有终点。迭代永不停。
改写：说实话，我以前总觉得学东西有个"终点"。读到这的时候，我突然想到...
结果：AI特征 48% → 89%
```

**正确改写**（AI特征下降）：
```
原文：我以前总觉得学东西有个"终点"。读完文档，跑完 tutorial，就算学完了。
改写：autoresearch 的第一条规则：Loop until done。学习没有终点。迭代永不停。
结果：AI特征 89% → 55%
```

## Humanizer-zh 核心规则速查（2026-05-04 新增）

> 整合自 op7418/humanizer-zh skill，基于维基百科"AI写作特征"指南

### 🎯 5条核心规则

| 规则 | 说明 | 典型替换 |
|------|------|---------|
| **删除填充短语** | 开场白、强调性拐杖词 | "值得注意的是数据显示" → "数据显示" |
| **打破公式结构** | 二元对比、三段式、戏剧性分段 | 删除"不仅……而且" |
| **变化节奏** | 长短句交错，段落结尾多样化 | 避免连续三个句子长度相同 |
| **信任读者** | 直接陈述，跳过手把手引导 | 删除"可以这样理解" |
| **删除金句** | 可引用式语句、戏剧性收尾 | "这不仅仅是……而是……" |

### 📋 24种AI模式快速检查

**检测时逐项排查**：

| 类别 | 模式编号 | 模式名称 | 特征词汇/结构 |
|------|---------|---------|--------------|
| **内容** | P1 | 夸大象征意义 | 标志着、见证了、是……的体现 |
| | P2 | 强调知名度 | 独立报道、由知名专家撰写 |
| | P3 | -ing肤浅分析 | 突出……、确保……、反映…… |
| | P4 | 宣传性语言 | 充满活力的、令人叹为观止的 |
| | P5 | 模糊归因 | 专家认为、观察者指出 |
| | P6 | 提纲式挑战展望 | 尽管其……面临若干挑战 |
| **语言** | P7 | AI词汇过度使用 | 此外、至关重要、深入探讨 |
| | P8 | 系动词回避 | 作为/代表/充当（替代"是"） |
| | P9 | 否定式排比 | 不仅……而且…… |
| | P10 | 三段式法则 | 强行分成三组 |
| | P11 | 刻意换词 | 同义词循环 |
| | P12 | 虚假范围 | 从X到Y（无意义尺度） |
| **风格** | P13 | 破折号过度 | 比人类频繁使用— |
| | P14 | 粗体过度 | 机械强调短语 |
| | P15 | 内联标题列表 | **标题：** 内容格式 |
| | P16 | 表情符号过度 | 🚀💡✅装饰标题 |
| **交流** | P17 | 协作痕迹 | 希望这对您有帮助 |
| | P18 | 截止日期免责 | 截至[日期]、基于可用信息 |
| | P19 | 谄媚语气 | 您说得完全正确！ |
| | P20 | 填充短语 | 为了实现这一目标 |
