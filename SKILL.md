# AI Content Detector Skill

## 功能定位

根据 Harrison Chase 的持续学习框架，第一步是**先把 traces 做对**。

本 Skill 实现：
1. **AI率检测**：模型辅助预判 + 朱雀官网验证（双保险）
2. **Traces记录**：记录检测历史，便于追溯和分析
3. **改写建议**：提供针对性改写建议
4. **阈值判断**：< 30% 通过，30-50% 警告，≥ 50% 不通过

**更新日志**：
- 2026-04-13：新增模型辅助预判，减少手动检测次数

---

## 核心工作流

### Step 1：接收输入

支持两种输入方式：
- **文件路径**：读取指定文件，提取正文（不含 YAML 元数据）
- **直接内容**：用户粘贴的文章内容

**优先级**：如果检测到文件路径，优先读取文件；否则使用粘贴内容。

---

### Step 2：模型辅助预判（NEW）

使用 LLM 进行快速分析，识别疑似AI生成段落。

**检测 prompt**（见下方 scripts/detect-prompt.md）

**输出格式**：

```markdown
## 🤖 模型辅助检测报告

### 整体评估
- **预估AI率**：XX%
- **风险等级**：🟢 低 / 🟡 中 / 🔴 高
- **核心问题**：[一句话概括]

### 高风险段落
#### 段落1 (位置：第X段)
> [原文片段]

**问题**：[具体原因]
**改写建议**：[针对性建议]

#### 段落2 ...

### 低风险段落
- [哪些段落看起来自然，可保留]

### 改写优先级
1. [最需要改的段落]
2. [次需要改的段落]
```

---

### Step 3：人工复核（可选）

根据模型预判结果，选择：
- **AI率 < 30%**：直接通过，记录 trace
- **AI率 30-50%**：建议人工改写后重新检测
- **AI率 ≥ 50%**：
  - 可选用朱雀官网二次验证
  - 或直接根据模型建议改写

朱雀检测指引（仅在需要时使用）：

```
检测工具：朱雀 AI 检测
官网地址：https://matrix.tencent.com/ai-detect/
```

---

### Step 4：阈值判断与建议

| AI率 | 状态 | 建议 |
|------|------|------|
| < 30% | ✅ 通过 | 记录 trace，继续流程 |
| 30-50% | ⚠️ 警告 | 根据模型建议改写，重新检测 |
| ≥ 50% | ❌ 不通过 | 必须大改，建议朱雀二次验证 |

---

### Step 5：记录 Traces

无论通过还是不通过，都记录检测历史，便于持续学习。

**traces 格式**：

```markdown
---
date: 2026-04-12
article: 文章标题（或内容前50字）
source: 文件路径 或 直接粘贴
tool: 模型辅助检测 + 朱雀验证
ai_rate: 45%
threshold: 30%
status: ⚠️ 警告
---

## 检测内容摘要
[文章核心内容摘要]

## 检测结果
- AI率：45%
- 判断：⚠️ 警告（需改写）

## 高风险段落
- 第3段："总而言之..."
- 第7段："值得注意的是..."

## 改写建议
- 第3段 → 替换为个人观点表达
- 第7段 → 删除过渡词

## 下一步行动
- 人工改写高 AI率段落
- 重新检测
```

**存储位置**：`~/.openclaw/workspace/ai-detection-traces/`
**文件命名**：`trace-{标题/日期}-{时间}.md`

## 使用方式

### 方式一：模型辅助检测（推荐）

**触发方式 A - 文件路径**：

```
AI检测 03-内容工厂/3-终稿确认区/终稿-Hermes记忆系统三条铁律-2026-04-12.md
```

**触发方式 B - 直接粘贴内容**：

```
AI检测
[直接粘贴文章内容]
```

或：

```
检测这段内容：
[文章内容]
```

**输出**：

```
## 🤖 模型辅助检测报告

### 整体评估
- **预估AI率**：42%
- **风险等级**：🟡 中
- **核心问题**：部分段落存在AI典型句式，缺乏个人经历细节

### 高风险段落

#### 第3段
> 总而言之，Hermes记忆系统的三条铁律为我们提供了一个清晰的框架...

**问题**：AI典型词汇"总而言之"、框架化表达
**改写建议**：替换为个人观点表达，如"从我使用 Hermes 的经验来看..."

#### 第7段
> 值得注意的是，这套系统并非完美无缺...

**问题**：AI典型词汇"值得注意的是"、过渡生硬
**改写建议**：直接表达观点，删除过渡词

### 低风险段落
第1段、第2段、第5段

### 改写优先级
1. 第3段 - AI率最高，影响整体判断
2. 第7段 - AI典型词汇明显

---

💡 建议：根据以上分析改写高风险段落后，可使用朱雀官网二次验证。
```

### 方式二：手动检测（备选）

**触发**：

```
朱雀检测 03-内容工厂/3-终稿确认区/终稿-xxx.md
```

**输出**：检测指引 + 内容，用户手动去朱雀官网检测
