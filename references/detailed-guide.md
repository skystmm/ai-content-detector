## 核心工作流

### Step 1：接收输入

- **文件路径**：读取指定文件，提取正文（不含 YAML 元数据）
- **直接内容**：用户粘贴的文章内容

优先级：文件路径优先，否则用粘贴内容。

### Step 2：36 种 AI 写作模式识别（v4）

使用 LLM 深度分析，基于 blader/humanizer v2.8.2（源自 Wikipedia "Signs of AI writing" 指南，WikiProject AI Cleanup 维护）。

#### 36 种模式速查表

| 类别 | 编号 | 模式名称 | 典型特征 |
|------|------|---------|---------|
| **内容** | P1 | 夸大象征意义 | 标志着、见证了、是……的体现 |
| | P2 | 强调知名度 | 独立报道、被NYT/FT报道 |
| | P3 | -ing肤浅分析 | 象征、反映、展示（含 -ing） |
| | P4 | 宣传性语言 | 坐落于、充满活力的、令人叹为观止 |
| | P5 | 模糊归因 | 专家认为、观察者指出 |
| | P6 | 提纲式挑战展望 | 尽管其……面临若干挑战 |
| **语言** | P7 | AI词汇过度使用 | 此外、至关重要、深入探讨、格局 |
| | P8 | 系动词回避 | 作为/代表/充当（替代"是"） |
| | P9 | 否定式排比 | 不仅……而且……；句尾否定碎片 |
| | P10 | 三段式法则 | 强行分成三组 |
| | P11 | 刻意换词（同义词循环） | 主人公→主要角色→英雄 |
| | P12 | 虚假范围 | 从X到Y（无意义尺度） |
| | P13 | 被动语态/无主语片段 | 无需配置文件、结果自动保留 |
| **风格** | P14 | 破折号硬约束 ❗ | 终稿禁止出现任何 — 或 – |
| | P15 | 粗体过度 | 机械强调短语 |
| | P16 | 内联标题列表 | **标题：** 内容格式 |
| | P17 | 标题大小写 | 所有主要词大写 |
| | P18 | 表情符号过度 | 🚀💡✅装饰标题 |
| | P19 | 花引号 | " " 替代 " " |
| | P20 | 连字符词组过度 | 跨职能的、基于数据的（谓语位置） |
| **交流** | P21 | 协作痕迹 | 希望对您有帮助、请告知 |
| | P22 | 截止免责/推测填空 | 截至某天、保持低调、很可能 |
| | P23 | 谄媚语气 | 您说得完全正确！ |
| | P24 | 填充短语 | 为了实现→为了、由于→因为 |
| | P25 | 过度限定 | 可以潜在地可能 |
| | P26 | 通用积极结论 | 未来看起来光明 |
| **修辞** | P27 | 说服性权威套路 | 真正的问题是、其核心是 |
| | P28 | 指示性宣告 | 让我们看看、以下是你需要知道的 |
| | P29 | 碎片化标题 | 标题后跟重复标题内容的一行段落 |
| | P30 | 差异锚定写作 | 仿佛叙述变更而非描述本身 |
| | P31 | 戏剧性短句堆叠 | 连续多个简短陈述句制造戏剧效果 |
| | P32 | 格言模板 | X是Y的语言、X成为陷阱 |
| | P33 | 对话式修辞开场 | 说实话？真正的问题是—— |
| | P34 | 刻意口语化（朱雀特例） | 完美口语化过渡=AI模仿 |

检测 prompt 见 `scripts/detect-prompt.md`。

### Step 3：改写流程（草稿 → 审计 → 终稿）

1. **草稿改写**：基于检测结果重写，保留信息密度（原文 5 段 → 改后 5 段）
2. **审计**：自问"上面这段为什么看起来还是像 AI 写的？"，标记剩余问题
3. **终稿**：解决审计问题，硬约束：终稿中无破折号（—/–）
4. **清理清单**（见 detect-prompt.md 末尾清单）

**改写原则**：

**❌ 错误策略**：过度简化、刻意口语化（"说实话"、"读到这"）、追求完美流畅

**✅ 正确策略**：保留技术术语原文、保持不完美表达（跳跃、简短）、删除整齐并列结构、注入个性（有观点、承认复杂性）、直接陈述勿宣告

### Step 4：阈值判断与建议

| AI率 | 状态 | 建议 |
|------|------|------|
| < 30% | ✅ 通过 | 记录 trace，继续流程 |
| 30-50% | ⚠️ 警告 | 根据模型建议改写，重新检测 |
| ≥ 50% | ❌ 不通过 | 必须大改，建议朱雀二次验证 |

### Step 5：记录 Traces

无论通过与否，都记录检测历史。

**traces 格式**：

```markdown
---
date: 2026-04-12
article: 文章标题（或内容前50字）
source: 文件路径 或 直接粘贴
tool: 模型辅助检测 + 朱雀验证
ai_rate: 45%
threshold: 30%
status: ⚠️ 警告
---

## 检测内容摘要
[文章核心内容摘要]

## 检测结果
- AI率：45%
- 判断：⚠️ 警告（需改写）

## 高风险段落
- 第3段："总而言之..."
- 第7段："值得注意的是..."

## 改写建议
- 第3段 → 替换为个人观点表达
- 第7段 → 删除过渡词

## 下一步行动
- 人工改写高 AI率段落
- 重新检测
```

**存储位置**：`~/.openclaw/workspace/ai-detection-traces/`
**文件命名**：`trace-{标题/日期}-{时间}.md`

## 误报防御

> 不要将以下人类写作特征误判为 AI

- ✅ 完美的语法和一致风格（专业作者）
- ✅ 混合正式和非正式语言（技术背景作者）
- ✅ "平淡"的文风（没有特定 AI 特征的一般性干涸）
- ✅ 信件格式的开头和结尾
- ✅ 单一过渡词（"然而"单独出现不是特征）
- ✅ 单一破折号（与其它特征组合才可疑）
- ✅ 一个简短强调句（连续多个堆叠才是问题）
- ✅ "说实话"句中正常使用（独立剧场式开场才是问题）
- ✅ 未打引用源的声明（网络大部分无引用）

**判断原则**：寻找特征的**组合簇**，而非孤立特征。

## 人类写作标志（Preserve These）

- 具体、不寻常、难以编造的细节
- 复杂感受和未解决的张力
- 有年代感的时代语言
- 句子长度变化
- 真实的插入语和自纠

## 朱雀检测实战经验

### 核心发现：朱雀判定逻辑与直觉相反

| 改写次数 | AI特征 | 改写策略 | 效果 |
|---------|--------|---------|------|
| 第一次 | 48.01% | 原文 | 基线 |
| 第二次 | 76.54% | 过度简化 | ❌ |
| 第三次 | 89.57% | 刻意口语化 | ❌ |
| 第四次 | 55.47% | 策略反转 | ✅ |
